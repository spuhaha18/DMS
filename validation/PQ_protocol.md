## 1. 목적 및 범위

본 프로토콜은 EDMS가 실제 운영 조건(파일럿 부서 사용자, 실제 업무 문서)에서 의도된 목적에 적합하게 동작함을 검증한다. OQ가 기능의 정확성을 검증했다면, PQ는 **실사용 시나리오에서의 통합 동작 적합성**을 검증한다.

PQ는 두 단계로 구성된다:
1. **데이터 이관 검증**: 활성 통제문서를 EDMS로 이관하고 데이터 정합성을 확인
2. **파일럿 운영 검증**: 파일럿 부서(QC팀 권장)가 1개월간 실무에서 시스템을 사용하며 검증

PQ 합격 후 시스템 릴리즈(Validation Report 작성 → QA 최종 승인)가 가능하다.

**PQ 수행 환경**: 운영 환경 (Production)  
**PQ 수행 주체**: QA + 파일럿 부서 사용자  
**OQ 선행 조건**: VAL-OQ-001 합격 서명 확인 후 PQ 착수

### 3.2 이관 프로세스

```
이관 절차:
1. QA가 이관 대상 문서 목록 작성 (metadata.csv)
2. 이관 담당자 2명이 metadata.csv 데이터 검증
3. Dry-run (이관 도구 테스트 환경 실행) → 오류 수정
4. Production Dry-run (실제 환경에서 읽기 전용 검증)
5. QA가 dry-run 결과 승인 → 실제 이관 실행 승인
6. edms-import 실행 → 이관 완료
7. QA가 이관 결과 샘플 검증 (PQ-MIG-002~005 수행)
8. QA 이관 승인 서명
```
