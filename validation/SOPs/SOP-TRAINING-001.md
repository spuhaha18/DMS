## 1. 목적

본 SOP는 EDMS 사용자의 신규 교육, 역할별 교육, 재교육 절차를 정의한다. Part 11 §11.10(i)는 전자기록·서명을 사용하는 인력이 적절히 교육받고 훈련받아야 함을 요구한다. 본 절차는 모든 EDMS 사용자가 시스템 사용 전 필요한 자격을 갖추도록 보장한다.

---

## 2. 범위

본 SOP는 EDMS에 접근하는 모든 사용자(내부 직원, 외부 감사인 포함)에게 적용된다.

---

## 3. 용어 정의

| 용어 | 정의 |
|---|---|
| 신규 교육 | EDMS 계정 활성화 전 처음 받는 교육 |
| 역할별 교육 | 배정된 역할(Author, Reviewer, Approver 등)에 특화된 실습 교육 |
| 재교육 | 연 1회 정기 또는 시스템 주요 변경 후 실시하는 교육 |
| 변경 교육 | Major 이상 시스템 변경 후 해당 기능 사용자에게 실시하는 교육 |
| Read & Acknowledge | 사용자가 문서를 열람하고 전자서명(ACKNOWLEDGED)으로 확인하는 행위 |
| 평가 | 교육 이해도를 확인하는 객관식 시험 (합격 기준: 80점 이상) |

---

| 역할 | 책임 |

| 시스템 관리자 | 교육 환경(sandbox) 제공, training_assignments 등록, 계정 활성화 |

| 사용자 | 교육 이수 및 평가 합격, 교육 기록 자가 확인 |

## 5. 절차

### 5.1 신규 계정 교육 — **계정 활성화 전 필수**

신규 계정 생성(SOP-USER-001 §5.1) 후 계정 활성화 전에 아래 교육을 완료해야 한다.

#### 5.1.1 교육 순서

```
Step 1: 이론 교육 (공통 과정)
  → EDMS 교육 자료 열람 + Read & Acknowledge (시스템 내)

Step 2: 역할별 실습 교육 (sandbox 환경)
  → 강사 또는 자기 주도 실습 (1~2시간)

Step 3: 평가
  → 객관식 20문항, 80점 이상 합격

Step 4: 교육 기록 서명
  → 교육 완료 확인서 Read & Acknowledge (ACKNOWLEDGED)

Step 5: 계정 활성화
  → 시스템 관리자가 training_assignments.completed_at 확인 후 계정 ACTIVE 전환
```

#### 5.1.2 공통 교육 내용 (전체 사용자, 약 1시간)

| 번호 | 내용 | 중요도 |
|---|---|---|
| 1 | EDMS 개요 및 GxP 전자기록·서명 규정 (Part 11, Annex 11) | 필수 |
| 2 | 로그인·비밀번호 관리·계정 공유 금지 | 필수 |
| 3 | 전자서명 의미와 법적 효력 | 필수 |
| 4 | 데이터 무결성 원칙 (ALCOA+) | 필수 |
| 5 | 이상 징후 신고 방법 | 필수 |
| 6 | 문서 검색 및 PDF 뷰어 사용 | 기본 |
| 7 | 알림센터 및 이메일 알림 | 기본 |

#### 5.1.3 역할별 추가 교육 내용

| 역할 | 추가 교육 내용 | 소요 시간 |
|---|---|---|
| **Author** | 문서 작성·채번·파일 업로드·PDF 미리보기·검토 요청 | 1시간 |
| **Reviewer** | 작업함 사용, 검토 의견 작성, REVIEWED 서명, 반려 절차 | 1시간 |
| **Approver** | 승인 판단 기준, APPROVED 서명 의미, qa_mandatory 흐름 | 1시간 |
| **QA** | 감사로그 조회, 권한 매트릭스 확인, 정기검토 응답 | 1시간 |
| **RA** | RA 특이 기능 (해당 시), 법규 해석 | 30분 |
| **READER** | 검색·열람만. 다운로드 정책 확인 | 30분 |
| **Admin** | 사용자 관리, 권한 매트릭스, 워크플로·채번 템플릿 관리, 감사로그 조회 | 2시간 |
| **Auditor** | 열람·감사로그 조회 전용 사용, 유효기간 인식 | 30분 |

### 5.2 평가 (Assessment)

- **형식**: 객관식 20문항 (4지선다)
- **합격 기준**: 80점(16문항) 이상
- **불합격 시**: 1회 재응시 가능 (24시간 후)
- **2회 연속 불합격 시**: 강사 1:1 보충 교육 후 3차 응시 가능
- **평가 기록**: 점수, 응시 일시, 합격 여부를 training_assignments에 기록

**평가 주요 항목**:
1. 전자서명 의미 (REVIEWED vs APPROVED 구분)
2. 계정 공유 금지 사유 (Part 11 §11.100 관련)
3. 비업무 시간 접근 이상 신고 절차
4. ALCOA+ 원칙 (특히 Contemporaneous — 서명 시점의 의미)
5. 워크플로 단계 이해 (제출 → 검토 → 승인 순서)
6. 비밀번호 분실 처리 방법

---

### 5.3 교육 기록 관리

training_assignments 테이블이 교육 이수의 공식 기록이다:

```
training_assignments:
  user_id: (사용자)
  version_id: (교육 자료 문서 버전)
  assigned_at: (교육 과제 배정일)
  due_at: (이수 기한)
  completed_at: (완료일 — 서명 시 자동 기록)
  completion_sig_id: (ACKNOWLEDGED 서명 ID — 증거)
```

- 교육 자료는 EDMS에 Form 카테고리로 등록하여 버전 관리
- 이수 확인: Read & Acknowledge (ACKNOWLEDGED 의미 전자서명)
- 완료 후 `training_assignments.completed_at`이 채워지면 이수 완료

---

### 5.4 재교육

| 재교육 유형 | 주기 | 대상 |
|---|---|---|
| 정기 재교육 | 연 1회 | 전체 사용자 |
| 변경 교육 | Major 이상 시스템 변경 후 | 영향받는 기능 사용자 |
| 직권 재교육 | 일탈 발생 후 | 관련 사용자 |

정기 재교육은 변경 사항 없더라도 핵심 내용(전자서명 의미, 계정 보안, 이상 신고) 복습을 포함한다.

---

### 5.5 분기 교육 현황 검토

**주기**: 분기 1회  
**수행자**: QA  
**목적**: 미이수자 현황 확인 및 독촉

```sql
-- 미이수 교육 과제 조회
SELECT u.user_id, u.full_name, u.department, ta.due_at
FROM training_assignments ta
JOIN users u ON u.id = ta.user_id
WHERE ta.completed_at IS NULL
  AND u.status = 'ACTIVE'
ORDER BY ta.due_at;
```

- 미이수자 명단을 부서 책임자에게 통보
- 기한 초과 사용자: 추가 7영업일 내 미이수 시 계정 잠금 검토 (QA Manager 승인)

---

### 5.6 Auditor 계정 교육

외부 감사인(Auditor 역할)은 시스템 접근 전 간략 교육을 이수한다:
- 열람·감사로그 조회 방법
- 다운로드 정책 (권한 없을 경우 다운로드 불가)
- 유효기간 내에만 접근 가능함을 확인
- 소요 시간: 약 30분

별도 sandbox 사용 불필요. 교육 기록은 부록 A에 수동으로 기록하거나 training_assignments에 등록.
