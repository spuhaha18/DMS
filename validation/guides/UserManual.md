## 1. 시작하기

### 1.1 로그인

1. 사내 PC에서 Chrome 또는 Edge 브라우저를 열고 `https://edms.lab.internal`에 접속합니다.
   - 이 주소는 사내망에서만 접속 가능합니다. VPN도 지원하지 않습니다.
2. 로그인 화면에서 사용자 ID와 비밀번호를 입력하고 **[로그인]** 버튼을 클릭합니다.

> **비밀번호 5회 오입력 시 계정이 30분 잠금**됩니다. 잠금 해제는 시스템 관리자에게 요청하세요.

### 1.2 첫 로그인 — 비밀번호 변경

신규 계정이거나 임시 비밀번호를 받은 경우, 첫 로그인 시 즉시 비밀번호 변경 화면이 열립니다.

비밀번호 규칙:
- 8자 이상
- 대문자·소문자·숫자·특수문자 중 3종 이상 포함
- 이전 5개 비밀번호와 동일 불가

### 1.3 대시보드

로그인 후 대시보드가 표시됩니다. 상단 메뉴:
- **문서**: 문서 목록·검색
- **내 작업**: 검토·승인 대기 목록
- **알림**: 알림센터
- **내 프로필**: 비밀번호 변경·교육 과제

## 2. 문서 검색 및 열람

### 2.1 기본 검색

1. 상단 메뉴 **[문서]** → **[검색]** 클릭, 또는 상단 검색창에 직접 입력합니다.
2. 검색어를 입력합니다 (최소 2자 이상). 한국어 형태소 검색을 지원합니다.
   - 예: "원료 수탁시험" → "원료", "수탁", "시험" 모두 포함된 문서 검색
3. 필터로 범위를 좁힐 수 있습니다:
   - **카테고리**: SOP / Method / Specification / Form
   - **부서**: 자신이 접근 권한을 가진 부서만 표시
   - **상태**: 시행중(EFFECTIVE) / 전체

### 2.2 검색 결과

검색 결과는 관련도 순으로 정렬됩니다. 각 행에는 문서 번호, 제목, 카테고리, 부서, 상태, 개정번호가 표시됩니다.

문서 번호를 클릭하면 문서 상세 화면으로 이동합니다.

### 2.3 PDF 뷰어

문서 상세 화면에서 **[열람]** 버튼을 클릭하면 PDF 뷰어가 열립니다.

| 기능 | 설명 |
|---|---|
| 확대/축소 | 뷰어 상단 +/- 버튼 또는 Ctrl+스크롤 |
| 페이지 이동 | 화살표 버튼 또는 직접 페이지 번호 입력 |
| 인쇄 | **다운로드 권한이 있는 경우만** 인쇄 버튼 활성화. 인쇄물에는 자동으로 "Uncontrolled When Printed" 표시 |
| 다운로드 | **다운로드 권한이 있는 경우만** 다운로드 버튼 활성화 |
| 서명 정보 | 뷰어 우측 패널 또는 서명 페이지(최종 페이지)에서 서명자·서명일시·의미 확인 |

> 다운로드 버튼이 보이지 않는다면 해당 문서에 대한 다운로드 권한이 없는 것입니다. 필요 시 부서 책임자를 통해 권한 요청을 하세요.

## 3. 문서 작성 (Author 역할)

### 3.1 문서 생성

1. **[문서]** → **[신규 문서]** 클릭
2. 아래 항목을 입력합니다:
   - **카테고리**: SOP / Method / Specification / Form
   - **부서**: 소속 부서 (자동 선택됨, 변경 가능)
   - **제목**: 문서 제목 (500자 이내)
   - **변경 사유**: 신규 작성 사유 (예: "신규 절차 수립")
3. **[생성]** 클릭 → 문서 번호가 자동으로 채번됩니다 (예: SOP-QC-001).
   - 채번된 번호는 변경할 수 없습니다.

### 3.2 파일 업로드

문서가 생성되면 초안(DRAFT) 상태입니다.

1. **[파일 업로드]** 버튼을 클릭합니다.
2. 작성한 문서 파일을 선택합니다 (지원 형식: `.docx`, `.xlsx`, `.pptx`, `.pdf`, 최대 100MB).
3. 업로드 후 PDF 자동 변환이 시작됩니다. 변환에는 1~5분이 소요됩니다.
4. 변환이 완료되면 **[미리보기]**로 PDF를 확인합니다.
   - 한글 폰트가 깨진 경우 시스템 관리자에게 문의하세요.
5. 내용 수정이 필요하면 파일을 다시 업로드합니다 (DRAFT 상태에서만 가능).

### 3.3 검토 요청

PDF 내용이 확인되면 검토 요청을 합니다.

1. **[검토 요청]** 버튼 클릭
2. 검토 요청 사유를 입력하고 **[확인]**
3. 문서 상태가 **검토중(UNDER_REVIEW)**으로 변경됩니다.
4. 워크플로에 설정된 검토자에게 자동으로 알림이 발송됩니다.

### 3.4 반려 처리

검토자가 반려하면 알림을 받습니다.

1. 알림에서 **[반려 사유 확인]** 클릭
2. 수정 후 파일을 다시 업로드합니다.
3. 다시 **[검토 요청]**을 합니다.

## 4. 검토 수행 (Reviewer 역할)

### 4.1 작업함 확인

1. 상단 **[내 작업]** 또는 대시보드 **[검토 대기]** 목록을 확인합니다.
2. 검토 요청된 문서 목록이 표시됩니다. 마감일이 가까운 순으로 정렬됩니다.

### 4.2 검토 수행

1. 검토할 문서를 클릭합니다.
2. **[열람]**에서 PDF 뷰어로 전체 내용을 검토합니다.
3. 검토 의견이 있으면 **[의견 추가]** 버튼으로 기록합니다 (선택).
4. 검토 완료 후:
   - 내용이 적절하면 → **[서명]** → 의미: REVIEWED 선택 → 비밀번호 입력 → **[서명 확인]**
   - 수정이 필요하면 → **[반려]** → 반려 사유 입력 → 비밀번호 입력 → **[반려 확인]**

> 서명은 다음 섹션 §6을 참조하세요.

---

## 5. 승인 수행 (Approver 역할)

검토 단계가 모두 완료되면 승인자에게 알림이 발송됩니다.

1. **[내 작업]** → **[승인 대기]** 목록에서 해당 문서를 클릭합니다.
2. **[열람]**에서 전체 내용 및 검토자 서명 내역을 확인합니다.
3. 승인 결정:
   - 승인 → **[서명]** → 의미: APPROVED 선택 → 비밀번호 입력 → **[서명 확인]**
   - 반려 → **[반려]** → 반려 사유 입력 → 비밀번호 입력 → **[반려 확인]**

> QA 역할의 최종 승인이 요구되는 카테고리(예: 일부 SOP)는 QA 승인자의 서명이 완료되면 **시행중(EFFECTIVE)** 상태로 자동 전환됩니다.

## 6. 전자서명

전자서명은 법적 효력이 있는 GxP 기록입니다. **신중하게** 사용하세요.

### 6.1 서명 절차

1. 해당 문서에서 **[서명]** 버튼을 클릭합니다.
2. 서명 다이얼로그가 열립니다:
   ```
   ┌────────────────────────────────────────┐
   │  전자서명 — SOP-QC-001 Rev.0           │
   ├────────────────────────────────────────┤
   │  서명 의미: [REVIEWED ▼]              │
   │  비밀번호: [____________]              │
   │  의견(선택): [                       ] │
   ├────────────────────────────────────────┤
   │  ■ 본인 [홍길동]으로서 위 의미에 따라  │
   │    전자적으로 서명함을 확인합니다.      │
   │  서명 일시: 2026-05-08 10:30:00 KST   │
   └────────────────────────────────────────┘
   ```
3. **서명 의미**를 확인합니다:
   - `REVIEWED` — 문서를 검토하고 내용이 적절함을 확인
   - `APPROVED` — 문서를 승인하여 시행을 허가
   - `QA_APPROVED` — QA 관점에서 승인
   - `ACKNOWLEDGED` — 내용을 열람·확인함
4. **비밀번호**를 입력합니다.
   - **세션 첫 서명**: 사용자 ID도 함께 입력해야 합니다 (Part 11 §11.200 요건).
   - **이후 서명**: 비밀번호만 입력합니다.
5. **[서명 확인]** 클릭.

### 6.2 서명 시 유의사항

- 서명 의미를 반드시 숙지하고 서명합니다. 잘못된 의미로 서명하면 취소할 수 없습니다.
- 실제 업무 행위(검토, 승인 등)가 완료된 후에만 서명합니다.
- 타인의 비밀번호로 서명하는 것은 Part 11 위반이며 징계 사유입니다.
- 서명 후에는 PDF 서명 페이지(마지막 페이지)에 서명 정보가 자동으로 반영됩니다.

## 7. 정기검토 응답 (문서 Owner)

본인이 Owner인 EFFECTIVE 문서의 정기검토 기한이 다가오면 알림을 받습니다.

1. **[알림센터]** 또는 이메일에서 정기검토 알림 클릭
2. 문서 상세 화면으로 이동 후 **[정기검토 응답]** 클릭
3. 검토 결과 선택:
   - **재확인** — 현행 문서 내용이 여전히 유효함. 다음 검토 주기로 연장
   - **개정** — 내용 수정이 필요함. 개정 절차 시작 (§3.1 문서 작성)
   - **폐기** — 문서가 더 이상 필요 없음. 폐기 처리
4. 선택 후 전자서명 (ACKNOWLEDGED)으로 응답 확정

> 정기검토 기한 알림 주기: 기한 90일, 30일, 7일, 0일 전. 기한 초과 시 매일 알림 (최대 30일).

## 8. 알림센터 및 이메일

### 8.1 앱 내 알림센터

상단 우측의 **알림 아이콘** (빨간 숫자)을 클릭하면 알림 목록이 표시됩니다.

| 알림 유형 | 발생 시점 |
|---|---|
| 검토 요청 | 문서가 검토 단계로 이동하고 내가 검토자인 경우 |
| 승인 요청 | 문서가 승인 단계로 이동하고 내가 승인자인 경우 |
| 반려 알림 | 내가 제출한 문서가 반려된 경우 |
| 시행 알림 | 내가 서명에 참여한 문서가 EFFECTIVE로 전환된 경우 |
| 정기검토 | 내가 Owner인 문서의 검토 기한 (90/30/7/0일 전) |
| 교육 과제 | 새로운 교육 과제가 배정된 경우 |

### 8.2 이메일 알림

동일한 알림이 등록된 회사 이메일 주소로도 발송됩니다. 이메일 내 링크를 클릭하면 해당 문서로 바로 이동합니다.

> 이메일을 받지 못하는 경우, 스팸 폴더를 확인하거나 시스템 관리자에게 문의하세요.

## 9. 내 교육 과제

1. 상단 우측 **[내 프로필]** → **[교육 과제]** 클릭
2. 배정된 교육 자료 목록이 표시됩니다.
3. 교육 자료를 클릭해 PDF 열람 후 하단의 **[열람 확인 서명]** 버튼으로 완료 처리합니다.
4. 평가가 있는 경우 **[평가 시작]** 버튼이 활성화됩니다 (80점 이상 합격).

## 10. FAQ 및 문제 신고

### 자주 묻는 질문

**Q. 비밀번호를 잊어버렸습니다.**  
A. 로그인 화면 하단의 **[비밀번호 초기화 요청]** 링크는 없습니다. 시스템 관리자에게 본인 확인 후 임시 비밀번호 발급을 요청하세요 (SOP-USER-001 §5.2.3).

**Q. 문서에 접근할 수 없습니다 (권한 없음).**  
A. 해당 문서의 카테고리 또는 부서에 대한 접근 권한이 없습니다. 부서 책임자를 통해 권한 요청을 하세요 (SOP-USER-001 §5.1).

**Q. PDF가 표시되지 않습니다 / 변환 중 오류가 발생했습니다.**  
A. 문서 상세 화면에서 PDF 상태가 `변환 실패`로 표시되면 시스템 관리자에게 문의하세요.

**Q. 잘못된 의미로 서명했습니다.**  
A. 전자서명은 취소할 수 없습니다. 부서 책임자와 QA에게 즉시 보고하세요 (SOP-DEVIATION-001).

**Q. 타인이 내 계정으로 접근한 것 같습니다.**  
A. 즉시 시스템 관리자에게 연락하여 계정을 잠가 달라고 요청하고, 비밀번호를 변경하세요. SOP-INCIDENT-001에 따라 보고됩니다.

### 문제 신고

시스템 오류, 이상 동작, 보안 의심 등은 다음 경로로 신고하세요:

| 신고 유형 | 신고 방법 |
|---|---|
| 일반 시스템 오류 | 시스템 관리자 이메일 또는 헬프데스크 티켓 |
| 보안 의심 사항 | 시스템 관리자 + 직속 부서 책임자에게 즉시 연락 |
| GxP 데이터 오류 | QA에게 즉시 보고 |
