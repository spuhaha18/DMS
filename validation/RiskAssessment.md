## 1. 목적 및 범위

본 문서는 EDMS Phase 1.a의 GxP 관련 기능에 대한 위험성 평가 결과를 기술한다. FMEA(Failure Mode and Effects Analysis) 방법론을 적용하여 각 기능의 잠재적 실패 모드, 영향, 원인 및 완화 조치를 식별한다.

평가 결과는 OQ/PQ 테스트 케이스의 우선순위 결정에 직접 활용된다. **Critical** 등급 기능은 OQ에서 반드시 검증되어야 하며, 검증 미완료 시 시스템 릴리즈가 불가하다.

**범위**: 사용자 패턴 데이터 신뢰성, 규제 준수, 환자 안전 및 제품 품질에 직접 영향을 미치는 GxP 관련 기능.

## 3. 위험성 평가 방법론

### 3.1 RPN (Risk Priority Number) 계산

```
RPN = 심각도(S) × 발생가능성(O) × 탐지가능성(D)
```

### 3.2 심각도(Severity) 척도

| 등급 | 점수 | 기준 |
|---|---|---|
| 치명적 | 5 | 환자 안전 위해, 규제 데이터 무결성 위반, 규제 제출 자료 훼손 |
| 높음 | 4 | 문서 데이터 손실, 전자서명 위조 가능성, 감사추적 훼손 |
| 보통 | 3 | 일부 기능 중단, 비인가 접근, 워크플로 지연 |
| 낮음 | 2 | UI 오류, 알림 누락, 성능 저하 |
| 미미 | 1 | 화면 표시 오류, 로그 누락(비GxP) |

### 3.3 발생가능성(Occurrence) 척도

| 등급 | 점수 | 기준 |
|---|---|---|
| 매우 높음 | 5 | 설계 결함 시 항상 발생 |
| 높음 | 4 | 운영 중 자주 발생 가능 |
| 보통 | 3 | 가끔 발생 |
| 낮음 | 2 | 드물게 발생 |
| 매우 낮음 | 1 | 거의 발생하지 않음 |

### 3.4 탐지가능성(Detectability) 척도

| 등급 | 점수 | 기준 |
|---|---|---|
| 탐지 불가 | 5 | 현재 제어 수단으로 탐지 불가 |
| 낮음 | 4 | 발생 후 상당 시간 후 탐지 |
| 보통 | 3 | 주기적 점검으로 탐지 |
| 높음 | 2 | 발생 직후 탐지 가능 |
| 매우 높음 | 1 | 자동 탐지(실시간 알림) |

### 3.5 위험 등급 기준

| 등급 | RPN 범위 | 조치 |
|---|---|---|
| **Critical** | RPN ≥ 50 또는 S = 5 | 반드시 완화 조치 구현 + OQ 검증 필수 |
| **High** | 25 ≤ RPN < 50 | 완화 조치 구현 + OQ 검증 권장 |
| **Medium** | 10 ≤ RPN < 25 | 완화 조치 검토 + OQ/PQ 검증 |
| **Low** | RPN < 10 | 수용 가능, 모니터링 |

## 7. 위험 완화 이후 검증 요건

### 7.1 IQ (Installation Qualification)에서 확인할 완화 조치

| 완화 조치 | IQ 체크 항목 |
|---|---|
| MinIO Object Lock COMPLIANCE 설정 | Object Lock mode = COMPLIANCE, retention = 3650일 확인 |
| NTP 동기화 설정 | chrony status, 사내 NTP 서버 연결 확인 |
| DB 역할 권한 분리 | audit_role에 UPDATE/DELETE 권한 없음 확인 |
| 한글 폰트팩 설치 | LibreOffice 변환 호스트에 Nanum/맑은고딕 폰트 존재 확인 |
| TLS 설정 | Nginx SSL 인증서, TLSv1.2+ 프로토콜, HSTS 헤더 확인 |

### 7.2 OQ (Operational Qualification)에서 필수 검증 항목

Critical 기능 21개에 대한 OQ 테스트 케이스는 `VAL-OQ-001` 프로토콜에서 정의된다. 각 Critical 기능은 최소 1개 이상의 양성(Pass) 케이스와 1개 이상의 음성(Reject) 케이스로 검증한다.

### 7.3 PQ (Performance Qualification)에서 확인할 완화 조치

| 완화 조치 | PQ 확인 항목 |
|---|---|
| 이관 메타데이터 정합성 | 파일럿 부서 이관 데이터 100% 샘플 검증 |
| 전체 워크플로 | 파일럿 부서 1개월 운영 — 비정상 전이 0건 확인 |
| 감사로그 완결성 | 파일럿 기간 액션 vs 로그 건수 일치 확인 |
