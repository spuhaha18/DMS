# ADR 0002 — 앵커 버킷은 보존정책 대상 외

| 항목 | 내용 |
|------|------|
| 날짜 | 2026-05-13 |
| 상태 | Accepted |
| 결정자 | IT/QA |

## 맥락

`AnchorService.java:79`는 Merkle 트리 감사 앵커를 MinIO `documents-anchors` 버킷(COMPLIANCE)에 올릴 때 3650일(10년)을 하드코딩으로 사용한다. M7.5의 `ProjectBasedRetentionResolver`를 여기에도 적용할지 검토가 필요하다.

## 결정

**앵커 버킷은 `ProjectBasedRetentionResolver` 적용 대상에서 제외한다.**

이유:
- 앵커는 연구과제와 무관한 audit 무결성 전용 아티팩트(SHA-256 해시 체인의 시간 증거).
- 앵커 보존기간 = 감사 기록 최장 보존기간과 동일해야 함(현재 10년).
- 연구과제별 보존기간(15~20년)이 앵커에 적용되면 앵커 보존이 과장될 수 있음.
- 두 정책의 독립 진화를 허용한다(앵커 정책은 향후 별도 ADR에서 다룸).

## 결과

- `AnchorService.java:79`의 `3650` 상수는 **변경하지 않는다**.
- M7.5 구현 중 이 줄을 건드리지 않도록 코드 주석으로 명시.
